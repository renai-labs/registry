# Report structures

Reusable skeletons. Pick the closest and trim — never pad a short ask into a long template.

## Briefing memo (default, ~1 page)

```
# <Title>: <one-line thesis>

## Summary
- <thesis in one sentence>
- <key number / fact 1> [1]
- <key number / fact 2> [2]
- <recommendation or "so what">

## <Section 1 — the main point>
<claim → evidence [n] → implication>

## <Section 2>
## <Section 3>

## Recommendation / Next steps
<what to do, given the above>

## Sources
[1] title — url (accessed YYYY-MM-DD)
```

## Market / competitive report

```
# <Market/space> — <timeframe>
## Executive summary        (thesis + 3–5 numbers that matter)
## Landscape                (size, growth, structure, key dynamics) [n]
## Players / themes         (one subsection each; consistent facets)
## Comparison               (table: player × dimension — render in xlsx if large)
## Outlook & risks
## Sources
```

## Literature / source summary

```
# <Question> — what the sources say
## Summary
## Themes                   (cluster findings; cite each)
## Agreement vs. disagreement
## Gaps / open questions
## Sources
```

## Decision doc

```
# Recommendation: <do X>
## Why (the case)
## Options considered       (A / B / C — pros, cons, cost)
## Risks & what must be true
## Sources
```

## Citation discipline

- Number sources in first-appearance order; reuse the same `[n]` on repeat reference.
- One canonical URL per source (strip tracking params).
- Every figure carries an "as of <date>".
- Distinguish sourced fact from your inference with explicit framing ("This implies…").
